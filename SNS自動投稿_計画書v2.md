## 1. プロジェクト概要

### 目的
画像や根拠となる情報を提供するだけで、Claude AI が記事を自動生成し、WordPress への投稿・Facebook・Instagram・TikTok・LIFULL介護への展開まで全工程を自動化する。

### 完成後のフロー（全体像）

```
担当者がソース情報を提供
（テキスト・URL・画像・PDF）
        ↓
  Make（自動化ハブ）
        ↓
  Claude API（記事生成）
  ├─ タイトル・本文・メタ情報の生成
  ├─ 提供画像のキャプション・ALTテキスト生成
  └─ SNS用の投稿文も同時生成
        ↓
  WordPress REST API（記事投稿）
        ↓
  HubSpot Webhook（記事公開を検知）
  ├─ Facebook（自動投稿）
  ├─ Instagram（自動投稿）
  └─ Make
       ├─ TikTok（半自動）
       └─ LIFULL介護（半自動）
```

## 3. Claude API による記事生成の詳細

### 3-1. 使用モデル

| 用途 | モデル | 理由 |
|------|--------|------|
| 記事本文の生成 | claude-sonnet-4-6 | 品質とコストのバランスが最適 |
| 画像解析・キャプション | claude-sonnet-4-6 | ビジョン機能で画像を直接解析 |
| SNS投稿文の生成 | claude-haiku-4-5 | 短文生成はコスト重視 |

### 3-2. 入力できるソース情報の種類

| 種類 | 内容 | 渡し方 |
|------|------|-------|
| テキスト | 伝えたい内容・箇条書きメモ | Google Driveのテキストファイル |
| 画像 | 商品・施設・イベント写真 | Google Driveのフォルダ |
| PDF | 資料・パンフレット | Google Drive |
| URL | 参考にするWebページ | テキストファイルに記載 |

### 3-3. Claude に生成させるコンテンツ

```
1. 記事タイトル（SEO最適化済み）
2. 記事本文（見出しH2/H3付き・約800〜1200文字）
3. メタディスクリプション（120文字以内）
4. アイキャッチ画像のALTテキスト
5. Facebook投稿文（300文字以内）
6. Instagram投稿文（ハッシュタグ付き・150文字以内）
7. TikTok用キャプション（100文字以内）
8. LIFULL介護投稿文（サービス特性に合わせたトーン）
```

### 3-4. 記事生成プロンプト設計（骨格）

```
あなたは介護業界の専門ライターです。
以下の情報をもとに、SEOを意識した記事を生成してください。

【提供情報】
{{ソーステキスト・画像解析結果・URL内容}}

【生成要件】
- ターゲット: 介護施設を探している家族・介護士を目指す方
- トーン: 親しみやすく専門的
- 文字数: 800〜1200文字
- 構成: リード文 → H2（3〜4つ） → まとめ
- SEOキーワード: {{キーワード}}

【出力形式】
タイトル:
本文（Markdown形式）:
メタディスクリプション:
Facebook投稿文:
Instagram投稿文（ハッシュタグ付き）:
TikTokキャプション:
LIFULL介護投稿文:
```

## 4. Make シナリオ設計（自動化フロー）

### シナリオ1：ソース受信 → 記事生成 → WordPress投稿

```
[トリガー]
Google Drive：指定フォルダに新規ファイルが追加された時

    ↓

[ファイル取得]
Make：テキスト・PDF・画像ファイルを読み込み

    ↓

[画像がある場合]
Claude API（Vision）：画像を解析してテキスト情報に変換

    ↓

[記事生成]
Claude API（claude-sonnet-4-6）：プロンプトに情報を渡して記事・SNS投稿文を一括生成

    ↓

[WordPress投稿]
WordPress REST API：
- ステータス：「下書き」として投稿（担当者が確認後に公開）
- タイトル・本文・メタ情報・画像を自動セット

    ↓

[担当者へ通知]
メール or Slack：「下書きが作成されました。確認してください」

    ↓

[担当者が確認・公開]
WordPress管理画面から公開ボタンを押す
```

### シナリオ2：WordPress公開 → SNS自動投稿

```
[トリガー]
WP Webhooks：WordPressで記事が公開された時

    ↓

[HubSpot]
Facebook・Instagram に自動投稿（生成済みの投稿文を使用）

    ↓

[Make]
担当者へ通知：TikTok・LIFULL介護の投稿を依頼
（生成済みのキャプション・投稿文を通知に添付）
```

## 5. WordPress REST API 設定

### 必要な設定

1. **アプリケーションパスワードの発行**
   - WordPress管理画面 → ユーザー → プロフィール → アプリケーションパスワード
   - Make から安全に認証するために使用

2. **投稿データの構造**
```json
{
  "title": "Claudeが生成したタイトル",
  "content": "Claudeが生成した本文（HTML変換済み）",
  "status": "draft",
  "meta": {
    "_yoast_wpseo_metadesc": "メタディスクリプション"
  },
  "featured_media": "アップロード済み画像のID"
}
```

3. **推奨プラグイン**
   - **WP Webhooks**：公開時にMake・HubSpotへ通知
   - **Yoast SEO**：メタ情報の管理（REST API対応）

## 9. リスクと対策

| リスク | 対策 |
|--------|------|
| Claude の生成内容が不正確 | 必ず担当者が確認してから「公開」する（下書きフロー） |
| WordPress REST API の認証エラー | アプリケーションパスワードを定期的に更新 |
| Make のシナリオ実行失敗 | エラー時はメール通知＋手動フォールバック手順を整備 |
| TikTok API の利用制限 | 手動投稿フローに即時切り替え |
| LIFULL介護が API 非対応 | 自動通知＋手動投稿で対応 |
