🗺️ システム全体概要
フロー B お知らせ → SNS自動投稿
  • 🕐 1
    Cloud Scheduler（30分ごと）
    POST /poll/news を自動呼び出し
  • 🔍 2
    aozora-cg.com/news/ をスクレイピング
    .post-title a セレクターで記事リンクを取得
  • 🔄 3
    新着チェック（重複防止）
    Cloud Storage の seen-urls.json と照合
  • 📄 4
    記事ページをスクレイピング
    h1.page-title / .entry-content / JSON-LD
  • 🤖 5
    Claude claude-sonnet-4-6 で投稿文生成
    Facebook / Instagram / TikTok 用に最適化
  • 📱 6
    HubSpot 経由で並列投稿
    Facebook & Instagram 同時投稿
  • 🎬 7
    Veo 3.1 で動画生成 → TikTok投稿
    縦型 9:16 / 15秒 / Content Posting API
  • 🔔 8
    Slack / メールで完了通知
    各プラットフォームの成功・失敗を報告
フロー A 素材 → WordPress下書き作成
  • 🕘 1
    Cloud Scheduler（毎朝9:00）
    POST /queue/process を自動呼び出し
  • 📂 2
    Google Drive から素材取得
    投稿素材_キュー フォルダの先頭フォルダ
  • 🖼️ 3
    Claude claude-sonnet-4-6 で画像解析
    添付画像の内容を日本語で説明
  • ✍️ 4
    Claude claude-sonnet-4-6 で記事生成
    SEO記事 + 各SNS投稿文を一括生成
  • 🎬 5
    Veo 3.1 で TikTok動画生成開始
    下書き作成と並列で実行
  • 📝 6
    WordPress に下書き保存
    SNS投稿文をカスタムフィールドにも保存
  • 🔔 7
    担当者へ確認通知
    公開はWordPress管理画面から手動で実施

📄 記事スクレイピング詳細
  • 📋
    一覧ページ
    https://aozora-cg.com/news/
    セレクター: .post-title a
  • 🏷️
    タイトル取得
    セレクター: h1.page-title
  • 📝
    本文取得
    セレクター: .entry-content
  • 🖼️
    アイキャッチ画像・抜粋
    JSON-LD (application/ld+json) から取得
    ない場合は本文冒頭120文字を使用
🤖 Claude プロンプト設計
  • 📘
    Facebook
    親しみやすい文体・URL付き・300文字以内
  • 📸
    Instagram
    共感系・絵文字・ハッシュタグ10個以上・150文字以内
  • 🎵
    TikTok
    短く引きつける一言・ハッシュタグ5個・100文字以内
  • 📤
    出力形式
    JSON形式で3種を一括出力
    パース失敗時はエラー通知
🛡️ エラーハンドリング
  • 🔁
    指数バックオフ リトライ
    5xx・429 エラー時に自動リトライ
    Facebook/Instagram: 最大3回
    TikTok/Claude: 最大2回
  • ⚡
    1件失敗しても継続
    新着が複数ある場合、1件エラーでも
    次の記事の処理を継続
  • 🎬
    TikTok動画生成失敗
    Veo 3.1 失敗時はTikTokのみスキップ
    Facebook・Instagramは通常投稿
  • 💾
    重複投稿防止
    投稿済みURLをCloud Storageで永続管理
    再起動・再デプロイ後も安全
