## 4. 最終的なシステム設計（v3.2）

### 2つのフロー

**フローA：新規記事作成**
```
Google Driveに素材投入（001・002...と番号順）
↓ 毎朝9:00 Cloud Schedulerが起動
↓ キューの先頭素材を1件取り出す
↓ Vertex AI Claude claude-sonnet-4-6 で記事・SNS投稿文を生成
↓ WordPress に下書き自動投稿
↓ 担当者が確認・公開（約5〜10分）
↓ フローBへ引き継ぎ
```

**フローB：既存記事からSNS展開**
```
WordPressで記事を公開（フローAでも直接書いた記事でもOK）
↓ WP Webhooks が即時通知
↓ WordPress REST API で記事内容を自動取得
↓ Claude が各SNS用投稿文を最適化・生成
↓ 全プラットフォームへ並列自動投稿
  ├─ Facebook（HubSpot経由）
  ├─ Instagram（HubSpot経由）
  ├─ TikTok（Veo 2で動画生成→Content Posting API）
  └─ LIFULL介護（パートナーAPI or Playwright）
```

### 使用技術スタック
| 技術 | 用途 |
|------|------|
| GCP Cloud Run | 自動化処理の中心 |
| Vertex AI Claude claude-sonnet-4-6 | 記事・SNS投稿文生成 |
| Vertex AI Veo 2 | TikTok用動画自動生成 |
| Cloud Scheduler | 毎朝9:00定期実行 |
| Cloud Pub/Sub | イベントキューイング |
| Secret Manager | APIキー管理 |
| Cloud Logging / Monitoring | ログ・監視 |
| HubSpot Marketing Starter | Facebook・Instagram管理 |
| WordPress REST API | 記事投稿・取得 |
| WP Webhooks（プラグイン） | 公開イベントの通知 |
| TikTok Content Posting API | TikTok投稿 |
| LIFULL介護 API / Playwright | LIFULL介護投稿 |
| Google Drive | 素材キュー管理 |
| rclone | ローカル→Google Drive同期 |
