---

## 1. やりたいこと（一言）

**Google Driveに素材を入れるだけで、AI が各SNSの投稿文・画像・動画を作り、担当者確認後に全SNSへ自動投稿。投稿後はWordPressのお知らせも自動更新する。**

---

## 2. 全体フロー

```
① 担当者がGoogle Driveの専用フォルダに素材を入れる
   （写真・テキスト・メモ など）
          ↓
② システムが素材を検知・取り込む
          ↓
③ AIが各SNS用のコンテンツを自動生成
   ├─ Facebook    : 投稿文
   ├─ Instagram   : 投稿文 ＋ リール動画
   ├─ TikTok      : リール動画 ＋ キャプション
   └─ X（Twitter）: 投稿文
          ↓
④ 担当者に確認依頼の通知が届く
   （生成されたコンテンツを確認・修正できる）
          ↓
⑤ 担当者が「承認」する
          ↓
⑥ 全SNSへ自動投稿
          ↓
⑦ WordPressのお知らせに記事を自動作成
   （担当者が内容を書いて公開）
```

---

## 3. 各ステップの詳細

### ① Google Drive 素材投入

フォルダ構成：
```
投稿素材_キュー/
├── 001_スタッフ紹介/
│   ├── 写真.jpg
│   └── コメント.txt     ← AIへの説明・指示を自由に記述
├── 002_イベント告知/
│   ├── 写真1.jpg
│   ├── 写真2.jpg
│   └── コメント.txt
├── 003_施設紹介/
│   └── コメント.txt     ← 画像なしでもOK
└── 処理済み/            ← 投稿後にシステムが自動移動
    └── 001_スタッフ紹介/
```

3つのルール：
| ルール | 内容 |
|--------|------|
| **順序** | フォルダ名の先頭に `001_` `002_` と番号をつける。小さい番号から処理 |
| **素材** | フォルダの中に画像（何枚でも可）と `コメント.txt` を入れる |
| **投稿済み** | 処理が終わったフォルダはシステムが `処理済み/` に自動移動 |

コメント.txt の書き方（例）：
```
今月入社したスタッフの山田さんの紹介です。
趣味はガーデニングで、利用者さんとの庭作りが得意。
明るくて元気な雰囲気が伝わるような投稿にしてください。
```

### ② 素材の検知・取り込み
- 毎朝9:00 に Cloud Scheduler がシステムを起動
- キュー内のフォルダを1件ずつ処理

### ③ AI によるコンテンツ生成
- **テキスト生成**：Claude（Anthropic）
  - 担当者が設定したプロンプトをもとに各SNS用の投稿文を生成
  - プラットフォームごとにトーン・文字数を調整
- **リール動画生成**：Veo（Google）
  - 写真とテキストから15〜60秒の動画を自動生成
  - Instagram・TikTok 用

### ④ 担当者への確認通知
- Slack または メール で通知
- 確認画面（シンプルなWebページ）で内容を確認・修正できる
- 投稿予定の全コンテンツが一覧表示される

### ⑤ 承認
- 確認画面の「承認」ボタンを押す
- 修正が必要な場合はテキストを直接編集して承認

### ⑥ SNS 自動投稿
| SNS | 投稿方法 | 投稿内容 |
|-----|---------|---------|
| Facebook | Meta Graph API | 投稿文 ＋ 画像 |
| Instagram | Meta Graph API | 投稿文 ＋ 画像 または リール動画 |
| TikTok | TikTok Content Posting API | リール動画 ＋ キャプション |
| X（Twitter）| X API v2 | 投稿文 ＋ 画像 |

### ⑦ WordPress お知らせ記事の自動作成
- SNS投稿完了後、WordPressに「下書き」を自動作成
- タイトルと本文の骨格（SNS投稿文をベース）を自動入力
- 担当者が内容を仕上げて「公開」ボタンを押す

## 4. システム構成（技術スタック）

```
【インフラ：Google Cloud Platform（GCP）】

Cloud Scheduler（毎朝9:00）
        ↓
Cloud Run（メインアプリ）
        ├─ Google Drive API    ← 素材の取得
        ├─ Claude API          ← 投稿文生成
        ├─ Veo（Vertex AI）   ← リール動画生成
        ├─ Cloud Storage      ← 生成動画の一時保存
        ├─ Firestore          ← 承認待ちデータの管理
        ├─ Meta Graph API     ← Facebook・Instagram投稿
        ├─ TikTok API         ← TikTok投稿
        ├─ X API v2           ← X（Twitter）投稿
        └─ WordPress REST API ← お知らせ下書き作成

Secret Manager           ← 全APIキーの安全な管理
Cloud Logging            ← 処理ログの記録
```

### 使用技術
| 項目 | 技術 |
|------|------|
| サーバー | Node.js / TypeScript |
| ホスティング | GCP Cloud Run |
| AI（テキスト） | Claude（Anthropic API） |
| AI（動画） | Veo（Google Vertex AI） |
| データ管理 | Firestore |
| 通知 | Slack Webhook または SendGrid（メール） |

## 5. SNS投稿文生成プロンプト設計

`draft-content` スキルの構造をベースに、Cloud Runアプリ用のプロンプトを設計する。

### プロンプトの構造

```
【役割】
あなたは介護施設の広報担当者です。
親しみやすく温かみのある文体で投稿文を作成してください。

【素材情報】
- コメント（担当者の説明）: {コメント.txtの内容}
- 画像の説明: {AIが解析した画像の内容}

【出力形式（JSON）】
{
  "facebook":  "300文字以内・絵文字あり・最後にCTA・ハッシュタグ3〜5個",
  "instagram": "150文字以内・絵文字あり・ハッシュタグ5〜10個",
  "tiktok":    "100文字以内・ハッシュタグ3〜5個",
  "x":         "140文字以内・絵文字あり・ハッシュタグ2〜3個"
}
```

### SNS別のルール

| SNS | 文字数 | 絵文字 | ハッシュタグ | その他 |
|-----|--------|--------|------------|--------|
| Facebook | 〜300文字 | あり | 3〜5個 | 最後に問い合わせCTAを入れる |
| Instagram | 〜150文字 | あり | 5〜10個 | ビジュアル重視の書き出し |
| TikTok | 〜100文字 | なし | 3〜5個 | 短くインパクト重視 |
| X | 〜140文字 | あり | 2〜3個 | 簡潔・リンクを想定 |

## 6. 生成物の管理・承認フロー

### 管理場所
| 生成物 | 保存場所 |
|--------|---------|
| SNS投稿文 | Googleスプレッドシート（自動書き出し） |
| リール動画 | Cloud Storage（URLをスプレッドシートに記載） |
| WordPress下書き | WordPress管理画面 |

### 承認フロー
```
AI生成完了
  ↓
Googleスプレッドシートに自動書き出し
（投稿文・動画URL・素材フォルダ名・生成日時）
  ↓
Google Chat Botが通知
「生成が完了しました。確認してください」
＋ スプレッドシートのリンク
＋「承認」「却下」ボタン
  ↓
担当者がスプレッドシートで内容確認・修正
  ↓
Google Chatの「承認」ボタンを押す
  ↓
全SNSへ自動投稿
```

### スプレッドシートの項目
| 列 | 内容 |
|----|------|
| 生成日時 | いつ生成されたか |
| 素材フォルダ名 | どの素材から生成したか |
| Facebook投稿文 | 生成されたテキスト（修正可） |
| Instagram投稿文 | 生成されたテキスト（修正可） |
| TikTok投稿文 | 生成されたテキスト（修正可） |
| X投稿文 | 生成されたテキスト（修正可） |
| 動画URL | Cloud StorageのURL |
| ステータス | 承認済み／却下／投稿済み |
| 投稿日時 | 実際に投稿された日時 |
