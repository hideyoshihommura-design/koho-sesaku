## 1. やりたいこと（一言）

**Google Chatに写真を送るだけで、AIが各SNSの投稿文・スライドショー動画を作り、担当者確認後に全SNSへ自動投稿。投稿後はWordPressのお知らせも自動更新する。**

---

## 2. 全体フロー

```
① 現場スタッフが既存のGoogle Chatグループに写真＋コメントを送る
   （今と同じ操作・新しいツール不要）
          ↓
② BotがGoogle Cloud Storage（GCS）に写真を自動保存
   同時にFirestoreに素材情報を登録（生成待ち状態）
          ↓
③ 毎日18:00・23:00 にAIが自動起動し各SNS用のコンテンツを生成
   ├─ Facebook    : 投稿文
   ├─ Instagram   : 投稿文 ＋ スライドショー動画（字幕・BGM付き）
   ├─ TikTok      : スライドショー動画 ＋ キャプション
   └─ X（Twitter）: 投稿文
          ↓
④ 担当者に確認依頼の通知が届く
   （Webアプリで投稿文・動画プレビューを確認・修正できる）
          ↓
⑤ 担当者が「承認」する
          ↓
⑥ 全SNSへ自動投稿
          ↓
⑦ WordPressのお知らせに記事を自動作成
   （担当者が内容を確認して公開）
```

## 3. 各ステップの詳細

### ① 素材の投入（現場スタッフ）

現場スタッフは**今と同じ操作**で素材を投入できる。新しいツールや手順は不要。

```
現場スタッフが既存のGoogle Chatグループに送る
├── 写真（何枚でも可）または 動画素材
└── コメント（AIへの説明・指示を自由に記述）

例）
「今月入社したスタッフの山田さんの紹介です。
　趣味はガーデニングで、利用者さんとの庭作りが得意。
　明るくて元気な雰囲気が伝わるような投稿にしてください。」
```

### ② 素材の検知・取り込み

#### Google Chat Bot による自動保存（リアルタイム）
- 既存のGoogle Chatグループ（全拠点共通）にBotを追加
- 現場スタッフがメッセージを送った瞬間にBotが検知
- **GCS（Google Cloud Storage）** に写真・動画を自動保存
- **Firestore** に素材情報を登録（生成待ち状態）
- 現場スタッフの操作は今と変わらない

**GCS のフォルダ構造（Botが自動生成）：**
```
GCSバケット（sns-videos）
├── materials/           ← スタッフが送った素材（入力）
│   └── 2026-04-02/
│       └── abc123/
│           ├── 1.jpg    ← 送った写真
│           ├── 2.jpg
│           └── 3.mp4   ← 送った動画素材（ある場合）
│
└── videos/              ← システムが生成したスライドショー（出力）
    └── abc123.mp4
```

#### 処理スケジュール（1日2回）

| 回 | 処理時刻 | 対象素材 |
|---|---|---|
| 1回目 | 18:00 | 主に14〜17時台に届いた素材 |
| 2回目 | 23:00 | 主に17〜22時台に届いた素材 |

- 翌朝には両バッチの処理が完了しており、出社後すぐ確認できる状態

### ③ AI によるコンテンツ生成

#### テキスト生成：Claude（Vertex AI 経由）
- コメントと画像をもとに各SNS用の投稿文を自動生成
- モデル：Claude Haiku（コスト最適化）
- 認証：Workload Identity（APIキー不要）

#### 動画生成：Remotion（無料・オープンソース）

| 項目 | 仕様 |
|------|------|
| 形式 | 縦型 9:16（1080×1920）Instagram Reels・TikTok対応 |
| 内容 | 写真のスライドショー |
| 1枚あたりの表示時間 | 5秒 |
| フェード効果 | 0.5秒 |
| 動画長さ（4枚の場合）| 約20秒 |
| 字幕 | Instagram投稿文の本文を画面下部に表示 |
| BGM | 3種類から自動でランダム選択（Dockerビルド時に自動生成） |

**BGMの種類：**
| ファイル | 和音 | 雰囲気 |
|---------|------|--------|
| bgm1.mp3 | Cメジャー | 明るい・爽やか |
| bgm2.mp3 | Fメジャー | 温かい・穏やか |
| bgm3.mp3 | Aマイナー | 落ち着き・やすらぎ |

#### 画像複数枚の扱い
| SNS | 扱い |
|---|---|
| Facebook | 投稿文のみ（画像はスタッフが手動投稿） |
| Instagram | スライドショー動画（字幕・BGM付き）|
| TikTok | スライドショー動画（キャプション付き）|
| X | 投稿文のみ |

#### 生成失敗時
- 自動で再試行（3回）
- 動画生成失敗時は投稿文のみ保存して続行（動画なしで承認画面に表示）
- 3回失敗後は担当者にエラー通知

### ④ 担当者への確認通知
- 生成完了後、Google Chat Botが承認用WebアプリのリンクつきでChat通知
- 3日経っても未対応の場合、再通知（リマインダー）

### ⑤ 確認・承認（Webアプリ）
- 担当者が承認ダッシュボード（Webアプリ）にアクセス
- 投稿文のプレビュー・生成動画のプレビューを確認
- 必要に応じて投稿文を直接編集
- SNSごとに個別に「承認」して投稿

### ⑥ SNS 自動投稿

| SNS | 投稿内容 |
|-----|---------|
| Facebook | 投稿文 ＋ 画像 |
| Instagram | 投稿文 ＋ スライドショー動画（字幕・BGM付き）|
| TikTok | スライドショー動画 ＋ キャプション |
| X（Twitter）| 投稿文 ＋ 画像（最大4枚） |

### ⑦ WordPress お知らせ記事の自動作成

- 承認済みSNSの投稿が完了した時点でWordPressに下書きを自動作成
- 担当者が確認して「公開」を押すだけ

## 4. システム構成（技術スタック）

```
【インフラ：Google Cloud Platform（GCP）】

Google Chat Bot（スタッフが写真を送る）
        ↓
Cloud Run（メインアプリ）
        ├─ GCS                ← 素材画像・生成動画の保存
        ├─ Firestore          ← 素材管理・承認状態の管理
        ├─ Vertex AI（Claude Haiku）← 投稿文生成
        ├─ Remotion           ← スライドショー動画生成（無料）
        ├─ Meta Graph API     ← Facebook・Instagram投稿
        ├─ TikTok API         ← TikTok投稿
        ├─ X API v2           ← X（Twitter）投稿
        └─ WordPress REST API ← お知らせ下書き作成

Cloud Scheduler（毎日18:00・23:00）← 処理の自動起動
Secret Manager   ← APIキーの安全な管理（3件のみ）
Cloud Logging    ← 処理ログの記録
```

### 使用技術
| 項目 | 技術 | 備考 |
|------|------|------|
| サーバー | Node.js / TypeScript | |
| ホスティング | GCP Cloud Run | CPU:2 / メモリ:2GB |
| AI（テキスト） | Claude Haiku（Vertex AI経由） | Workload Identity認証 |
| AI（動画） | Remotion（React）| 無料・オープンソース |
| BGM | ffmpeg自動生成 | Dockerビルド時に自動作成 |
| 素材・動画保存 | GCS（Cloud Storage）| 1バケットで統一管理 |
| データ管理 | Firestore | |
| 通知 | Google Chat Webhook | |
| 認証 | Workload Identity（ADC）| APIキー管理が最小限 |

### Secret Manager の登録内容（3件のみ）
| シークレット名 | 内容 |
|--------------|------|
| chat-webhook-url | 通知用 Google Chat Incoming Webhook URL |
| app-secret-path | 承認WebアプリのURL秘密パス |
| scheduler-secret | Cloud Scheduler 認証トークン |

## 5. SNS投稿文生成プロンプト設計

### プロンプトの構造

```
【役割】
あなたは介護施設の広報担当者です。
親しみやすく温かみのある文体で投稿文を作成してください。

【素材情報】
- コメント（担当者の説明）: {コメントの内容}
- 添付画像: {AIが解析した画像の内容}

【出力形式（JSON）】
{
  "facebook":  "300文字以内・絵文字あり・最後にCTA・ハッシュタグ3〜5個",
  "instagram": "150文字以内・絵文字あり・ハッシュタグ5〜10個",
  "tiktok":    "100文字以内・ハッシュタグ3〜5個",
  "x":         "140文字以内・絵文字あり・ハッシュタグ2〜3個"
}
```

### SNS別のルール

| SNS | 文字数 | 絵文字 | ハッシュタグ | その他 |
|-----|--------|--------|------------|--------|
| Facebook | 〜300文字 | あり | 3〜5個 | 最後に問い合わせCTAを入れる |
| Instagram | 〜150文字 | あり | 5〜10個 | ビジュアル重視の書き出し・動画字幕にも使用 |
| TikTok | 〜100文字 | なし | 3〜5個 | 短くインパクト重視 |
| X | 〜140文字 | あり | 2〜3個 | 簡潔・リンクを想定 |

## 6. 生成物の管理・承認フロー

### 管理場所
| 生成物 | 保存場所 |
|--------|---------|
| SNS投稿文・承認状態 | Firestore |
| スタッフが送った写真・動画 | GCS（materials/フォルダ）|
| 生成されたスライドショー動画 | GCS（videos/フォルダ）|
| WordPress下書き | WordPress管理画面 |

### 承認フロー
```
AI生成完了（投稿文 ＋ スライドショー動画）
  ↓
Firestoreに保存（generationStatus: generated）
Google Chat Botが承認WebアプリのURLを通知
  ↓
担当者が承認Webアプリを開いて確認・修正
  ├─ 投稿文のプレビュー・編集
  └─ 動画のプレビュー（GCS署名付きURL・1時間有効）
  ↓
【肖像権確認】hasFacePermissionフラグを確認
  ├─ OK → そのまま承認へ
  └─ 要確認 → 担当者が判断してから承認
  ↓
SNSごとに「承認」ボタンで判断
  ↓
「承認」したSNSのみ自動投稿

※ 3日経っても未対応の場合、Google Chatで再通知
```
