## 1. やりたいこと（一言）

**Google Driveに素材を入れるだけで、AI が各SNSの投稿文・画像・動画を作り、担当者確認後に全SNSへ自動投稿。投稿後はWordPressのお知らせも自動更新する。**

---

## 2. 全体フロー

```
① 担当者がGoogle Driveの専用フォルダに素材を入れる
   （写真・テキスト・メモ など）
          ↓
② システムが素材を検知・取り込む
          ↓
③ AIが各SNS用のコンテンツを自動生成
   ├─ Facebook    : 投稿文
   ├─ Instagram   : 投稿文 ＋ リール動画
   ├─ TikTok      : リール動画 ＋ キャプション
   └─ X（Twitter）: 投稿文
          ↓
④ 担当者に確認依頼の通知が届く
   （生成されたコンテンツを確認・修正できる）
          ↓
⑤ 担当者が「承認」する
          ↓
⑥ 全SNSへ自動投稿
          ↓
⑦ WordPressのお知らせに記事を自動作成
   （担当者が内容を書いて公開）
```

---

## 3. 各ステップの詳細

### ① Google Drive 素材投入

フォルダ構成：
```
投稿素材_キュー/
├── 001_スタッフ紹介/
│   ├── 写真.jpg
│   └── コメント.txt     ← AIへの説明・指示を自由に記述
├── 002_イベント告知/
│   ├── 写真1.jpg
│   ├── 写真2.jpg
│   └── コメント.txt
├── 003_施設紹介/
│   └── コメント.txt     ← 画像なしでもOK
└── 処理済み/            ← 投稿後にシステムが自動移動
    └── 001_スタッフ紹介/
```

3つのルール：
| ルール | 内容 |
|--------|------|
| **順序** | フォルダ名の先頭に `001_` `002_` と番号をつける。小さい番号から処理 |
| **素材** | フォルダの中に画像（何枚でも可）と `コメント.txt` を入れる |
| **投稿済み** | 処理が終わったフォルダはシステムが `処理済み/` に自動移動 |

コメント.txt の書き方（例）：
```
今月入社したスタッフの山田さんの紹介です。
趣味はガーデニングで、利用者さんとの庭作りが得意。
明るくて元気な雰囲気が伝わるような投稿にしてください。
```

### ② 素材の検知・取り込み
- 毎朝9:00 に Cloud Scheduler がシステムを起動
- キュー内のフォルダを1件ずつ処理

### ③ AI によるコンテンツ生成
- **テキスト生成**：Claude（Anthropic）
  - 担当者が設定したプロンプトをもとに各SNS用の投稿文を生成
  - プラットフォームごとにトーン・文字数を調整
- **リール動画生成**：Veo（Google）
  - 写真とテキストから15〜60秒の動画を自動生成
  - Instagram・TikTok 用

### ④ 担当者への確認通知
- Slack または メール で通知
- 確認画面（シンプルなWebページ）で内容を確認・修正できる
- 投稿予定の全コンテンツが一覧表示される

### ⑤ 承認
- 確認画面の「承認」ボタンを押す
- 修正が必要な場合はテキストを直接編集して承認

### ⑥ SNS 自動投稿
| SNS | 投稿方法 | 投稿内容 |
|-----|---------|---------|
| Facebook | Meta Graph API | 投稿文 ＋ 画像 |
| Instagram | Meta Graph API | 投稿文 ＋ 画像 または リール動画 |
| TikTok | TikTok Content Posting API | リール動画 ＋ キャプション |
| X（Twitter）| X API v2 | 投稿文 ＋ 画像 |

### ⑦ WordPress お知らせ記事の自動作成
- SNS投稿完了後、WordPressに「下書き」を自動作成
- タイトルと本文の骨格（SNS投稿文をベース）を自動入力
- 担当者が内容を仕上げて「公開」ボタンを押す

## 4. システム構成（技術スタック）

```
【インフラ：Google Cloud Platform（GCP）】

Cloud Scheduler（毎朝9:00）
        ↓
Cloud Run（メインアプリ）
        ├─ Google Drive API    ← 素材の取得
        ├─ Claude API          ← 投稿文生成
        ├─ Veo（Vertex AI）   ← リール動画生成
        ├─ Cloud Storage      ← 生成動画の一時保存
        ├─ Firestore          ← 承認待ちデータの管理
        ├─ Meta Graph API     ← Facebook・Instagram投稿
        ├─ TikTok API         ← TikTok投稿
        ├─ X API v2           ← X（Twitter）投稿
        └─ WordPress REST API ← お知らせ下書き作成

Secret Manager           ← 全APIキーの安全な管理
Cloud Logging            ← 処理ログの記録
```

### 使用技術
| 項目 | 技術 |
|------|------|
| サーバー | Node.js / TypeScript |
| ホスティング | GCP Cloud Run |
| AI（テキスト） | Claude（Anthropic API） |
| AI（動画） | Veo（Google Vertex AI） |
| データ管理 | Firestore |
| 通知 | Slack Webhook または SendGrid（メール） |

---

## 5. 確認画面（承認UI）の仕様

- シンプルなWebページとして Cloud Run 上に同居
- 投稿予定のコンテンツが一覧表示される
- 各SNSの投稿文をその場で編集できる
- 「全て承認」「1つだけ承認」「却下」のボタン
- 承認後、自動的に全SNSへ投稿が実行される
