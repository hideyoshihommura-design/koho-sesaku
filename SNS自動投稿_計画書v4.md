## 1. やりたいこと（一言）

**Google Driveに素材を入れるだけで、AI が各SNSの投稿文・画像・動画を作り、担当者確認後に全SNSへ自動投稿。投稿後はWordPressのお知らせも自動更新する。**

---

## 2. 全体フロー

```
① 現場スタッフが既存のGoogle Chatグループに写真＋コメントを送る
   （今と同じ操作・新しいツール不要）
          ↓
② BotがGoogle Driveに自動保存・システムが素材を取り込む
          ↓
③ AIが各SNS用のコンテンツを自動生成
   ├─ Facebook    : 投稿文
   ├─ Instagram   : 投稿文 ＋ リール動画
   ├─ TikTok      : リール動画 ＋ キャプション
   └─ X（Twitter）: 投稿文
          ↓
④ 担当者に確認依頼の通知が届く
   （生成されたコンテンツを確認・修正できる）
          ↓
⑤ 担当者が「承認」する
          ↓
⑥ 全SNSへ自動投稿
          ↓
⑦ WordPressのお知らせに記事を自動作成
   （担当者が内容を書いて公開）
```

---

## 3. 各ステップの詳細

### ① 素材の投入（現場スタッフ）

現場スタッフは**今と同じ操作**で素材を投入できる。新しいツールや手順は不要。

```
現場スタッフが既存のGoogle Chatグループに送る
├── 写真（何枚でも可）
└── コメント（AIへの説明・指示を自由に記述）

例）
「今月入社したスタッフの山田さんの紹介です。
　趣味はガーデニングで、利用者さんとの庭作りが得意。
　明るくて元気な雰囲気が伝わるような投稿にしてください。」
```

Google Drive上のフォルダ構成（Botが自動生成）：
```
投稿素材_キュー/
├── 001_20260401_0912/   ← 受け取った順に番号・日時で自動命名
│   ├── 写真.jpg
│   └── コメント.txt
├── 002_20260401_1034/
│   ├── 写真1.jpg
│   ├── 写真2.jpg
│   └── コメント.txt
└── 処理済み/            ← 投稿後にシステムが自動移動
    └── 001_20260401_0912/
```

- フォルダの作成・番号付けはすべて自動
- このグループは素材専用のため、すべてのメッセージを素材として処理

### ② 素材の検知・取り込み

1日2回処理する。素材の送信が多い時間帯（14〜17時）に合わせたスケジュール：

| 回 | 処理時刻 | 通知タイミング | 対象素材 |
|---|---|---|---|
| 1回目 | 18:00 | 18時過ぎ | 主に14〜17時台に届いた素材 |
| 2回目 | 23:00 | 翌朝8:00 | 主に17〜20時台に届いた素材 |

- 翌朝9:00には両バッチの通知が届いており、出社後すぐ確認できる状態になっている
- キュー内のフォルダをすべて処理（複数件まとめて処理）

### ③ AI によるコンテンツ生成
- **テキスト生成**：Claude（Anthropic）
  - 担当者が設定したプロンプトをもとに各SNS用の投稿文を生成
  - プラットフォームごとにトーン・文字数を調整
- **リール動画生成**：Veo（Google）
  - 写真とテキストから15〜60秒の動画を自動生成
  - Instagram・TikTok 用

### ④ 担当者への確認通知
- Google Chat BotがスプレッドシートのリンクつきでGoogle Chatに通知
- 3日経っても未対応の場合、Google Chatで再通知（リマインダー）

### ⑤ 確認・承認
- 担当者がスプレッドシートを開いて内容確認
- 必要に応じてセルを直接編集（修正した内容がそのまま投稿される）
- SNSごとに個別に「投稿する」「投稿しない」ボタンで判断
- 承認者は誰か1人でOK（複数人でも最初に押した人が確定）

### ⑥ SNS 自動投稿
| SNS | 投稿方法 | 投稿内容 |
|-----|---------|---------|
| Facebook | Meta Graph API | 投稿文 ＋ 画像 |
| Instagram | Meta Graph API | 投稿文 ＋ 画像 または リール動画 |
| TikTok | TikTok Content Posting API | リール動画 ＋ キャプション |
| X（Twitter）| X API v2 | 投稿文 ＋ 画像 |

### ⑦ WordPress お知らせ記事の自動作成
- SNS投稿完了後、WordPressに「下書き」を自動作成
- タイトルと本文の骨格（SNS投稿文をベース）を自動入力
- 担当者が内容を仕上げて「公開」ボタンを押す

## 4. システム構成（技術スタック）

```
【インフラ：Google Cloud Platform（GCP）】

Cloud Scheduler（毎朝9:00）
        ↓
Cloud Run（メインアプリ）
        ├─ Google Drive API    ← 素材の取得
        ├─ Claude API          ← 投稿文生成
        ├─ Veo（Vertex AI）   ← リール動画生成
        ├─ Cloud Storage      ← 生成動画の一時保存
        ├─ Firestore          ← 承認待ちデータの管理
        ├─ Meta Graph API     ← Facebook・Instagram投稿
        ├─ TikTok API         ← TikTok投稿
        ├─ X API v2           ← X（Twitter）投稿
        └─ WordPress REST API ← お知らせ下書き作成

Secret Manager           ← 全APIキーの安全な管理
Cloud Logging            ← 処理ログの記録
```

### 使用技術
| 項目 | 技術 |
|------|------|
| サーバー | Node.js / TypeScript |
| ホスティング | GCP Cloud Run |
| AI（テキスト） | Claude（Anthropic API） |
| AI（動画） | Veo（Google Vertex AI） |
| データ管理 | Firestore |
| 通知 | Slack Webhook または SendGrid（メール） |

## 5. SNS投稿文生成プロンプト設計

`draft-content` スキルの構造をベースに、Cloud Runアプリ用のプロンプトを設計する。

### プロンプトの構造

```
【役割】
あなたは介護施設の広報担当者です。
親しみやすく温かみのある文体で投稿文を作成してください。

【素材情報】
- コメント（担当者の説明）: {コメント.txtの内容}
- 画像の説明: {AIが解析した画像の内容}

【出力形式（JSON）】
{
  "facebook":  "300文字以内・絵文字あり・最後にCTA・ハッシュタグ3〜5個",
  "instagram": "150文字以内・絵文字あり・ハッシュタグ5〜10個",
  "tiktok":    "100文字以内・ハッシュタグ3〜5個",
  "x":         "140文字以内・絵文字あり・ハッシュタグ2〜3個"
}
```

### SNS別のルール

| SNS | 文字数 | 絵文字 | ハッシュタグ | その他 |
|-----|--------|--------|------------|--------|
| Facebook | 〜300文字 | あり | 3〜5個 | 最後に問い合わせCTAを入れる |
| Instagram | 〜150文字 | あり | 5〜10個 | ビジュアル重視の書き出し |
| TikTok | 〜100文字 | なし | 3〜5個 | 短くインパクト重視 |
| X | 〜140文字 | あり | 2〜3個 | 簡潔・リンクを想定 |

## 6. 生成物の管理・承認フロー

### 管理場所
| 生成物 | 保存場所 |
|--------|---------|
| SNS投稿文 | Googleスプレッドシート（自動書き出し） |
| リール動画 | Cloud Storage（URLをスプレッドシートに記載） |
| WordPress下書き | WordPress管理画面 |

### 承認フロー
```
AI生成完了
  ↓
Googleスプレッドシートに自動書き出し
（投稿文・動画URL・素材フォルダ名・生成日時）
  ↓
Google Chat Botが通知（スプレッドシートのリンクのみ）
「生成が完了しました。確認してください」
  ↓
担当者がスプレッドシートを開いて内容確認・修正（セルを直接編集）
  ↓
SNSごとに「投稿する」「投稿しない」ボタンで判断
  ↓
「投稿する」にしたSNSのみ自動投稿

※ 3日経っても未対応の場合、Google Chatで再通知
※ 「投稿しない」にしても行は削除せず履歴として残す
```

### スプレッドシートの項目
| 列 | 内容 |
|----|------|
| 生成日時 | いつ生成されたか |
| 素材フォルダ名 | どの素材から生成したか |
| 拠点 | コメントからClaudeが自動検出（不明の場合は手動入力） |
| Facebook投稿文 | 生成されたテキスト（修正可） |
| Facebook | 投稿する／投稿しない |
| Instagram投稿文 | 生成されたテキスト（修正可） |
| Instagram | 投稿する／投稿しない |
| TikTok投稿文 | 生成されたテキスト（修正可） |
| TikTok | 投稿する／投稿しない |
| X投稿文 | 生成されたテキスト（修正可） |
| X | 投稿する／投稿しない |
| 動画URL | Cloud StorageのURL |
| 投稿日時 | 実際に投稿された日時 |

### ステータスの種類
| ステータス | 意味 |
|---|---|
| 確認待ち | 生成直後、まだ確認していない |
| 承認済み | 「投稿する」ボタンを押した |
| 投稿しない | 今回はスキップ |
| 投稿済み | SNS投稿完了 |
